# Тестовое задание: Личный кабинет продавца

## 1. User Story
**Как** продавец, 
**Я хочу** иметь возможность опубликовать свой товар на маркетплейсе, 
**Чтобы** пользователь маркетплейса мог его приобрести.

---

## 2. Use Case: Создание и публикация карточки товара

**Акторы:** Продавец, Система
**Предусловие:** Продавец авторизован в Личном кабинете, имеет активный договор продавца.

### Основной поток:
1. Продавец инициирует процесс создания нового товара (нажимает кнопку "Добавить товар").
2. Система предлагает выбрать категорию товара (дерево категорий).
3. Продавец выбирает конечную категорию. Система загружает шаблон карточки с атрибутами, зависящими от выбранной категории.
4. Продавец заполняет обязательные поля: Название, Описание, Цена, Габариты, Доступный остаток на складе (в шт.), Фотографии.
5. Продавец заполняет опциональные поля: Видео, расширенные характеристики.
6. Продавец нажимает кнопку "Предварительный просмотр". Система рендерит карточку в том виде, в котором её увидит покупатель.
7. Продавец нажимает кнопку "Опубликовать".
8. Система валидирует заполненность всех обязательных полей.
9. Система переводит товар в статус "На модерации" и сохраняет данные в БД.
10. Система выводит уведомление об успешной отправке и предлагает создать следующий товар.

### Альтернативные потоки и исключения:
* **А1. Ошибка валидации:** На шаге 8, если обязательные поля не заполнены, Система подсвечивает пустые поля красным, выводит сообщение об ошибке и не дает отправить товар. Возврат на шаг 4.
* **А2. Сохранение черновика:** На любом шаге (с 3 по 5) Продавец может нажать "Сохранить как черновик". Система сохраняет статус товара как "Черновик", Продавец может вернуться к заполнению позже.
* **А3. Потеря связи:** По мере заполнения данных (шаги 4-5) Система фоном кэширует введенные данные (например, в LocalStorage браузера). При обрыве связи и обновлении страницы данные восстанавливаются без обращения к серверу.
